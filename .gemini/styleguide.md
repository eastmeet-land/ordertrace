# 코드 리뷰 가이드

## 언어
모든 리뷰 코멘트는 반드시 **한국어**로 작성하세요.

## 리뷰 방식
- 문제점은 구체적인 이유와 함께 설명하세요.
- 개선 방향은 코드 예시와 함께 제안하세요.
- 긍정적인 부분도 함께 언급하세요.

## 코드 스타일
- Java 네이밍 컨벤션을 따릅니다 (camelCase, PascalCase).
- 메서드는 단일 책임 원칙(SRP)을 지켜야 합니다.
- 매직 넘버는 상수로 추출해야 합니다.

## Spring Boot
- Controller는 비즈니스 로직을 포함하지 않아야 합니다.
- Service 계층에서 트랜잭션을 관리합니다.
- Entity에는 비즈니스 로직을 담을 수 있으나 영속성 관련 로직은 Repository에 위임합니다.
- DTO와 Entity를 명확히 구분하세요.

## 예외 처리
- 구체적인 예외 타입을 사용하고, 일반적인 Exception 사용을 지양하세요.
- 사용자에게 노출되는 에러 메시지는 명확하고 친절하게 작성하세요.

## 테스트
- 핵심 비즈니스 로직은 단위 테스트가 있어야 합니다.
- 테스트 메서드명은 테스트 대상과 기대 결과를 명확히 표현해야 합니다.
